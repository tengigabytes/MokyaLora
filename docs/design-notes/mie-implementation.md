## 3. Binary Data Formats

### 3.1 `dict_dat.bin` Рђћ MIED Format

All multi-byte fields are **little-endian**.

```
Header (16 bytes):
  Offset  Size  Field
   0       4    magic         = "MIED"  (0x4D 0x49 0x45 0x44)
   4       2    version       = 1
   6       2    flags         = 0
   8       4    key_count     Рђћ number of unique Bopomofo keys
  12       4    keys_data_off Рђћ byte offset of the keys-data section

Index table (key_count ├Ќ 8 bytes, immediately after the header):
  Entries are sorted in lexicographic byte order of the corresponding key.
  Offset  Size  Field
   0       4    key_data_off  Рђћ byte offset of this key within the keys-data section
   4       4    val_data_off  Рђћ byte offset of the ValueRecord in dict_values.bin

Keys-data section (at keys_data_off, variable-length records):
  Offset  Size  Field
   0       1    key_len       Рђћ UTF-8 byte length of the Bopomofo key
   1      N     key_utf8      Рђћ key_len bytes (NOT null-terminated)
```

**Search complexity:** O(log N) binary search over the fixed-width index table,
with one random-access read into the keys-data section per probe.

### 3.2 `dict_values.bin` Рђћ ValueRecord

```
ValueRecord at val_data_off:
  Offset  Size  Field
   0       2    word_count    Рђћ number of candidate words
  Per word (sorted by frequency descending):
   0       2    freq          Рђћ frequency weight (higher = more common)
   2       1    word_len      Рђћ UTF-8 byte length of the word
   3      N     word_utf8     Рђћ word_len bytes (NOT null-terminated)
```

### 3.3 `font_glyphs.bin` / `font_index.bin`

Generated by `tools/gen_font.py`.  Requires a GNU Unifont `.hex` source file
and the MoE 8,104-character list.

| File | Format | Size |
|------|--------|------|
| `font_glyphs.bin` | 16 ├Ќ 16 bitmap, 32 bytes / glyph, packed | ~260 KB |
| `font_index.bin` | `(codepoint: uint32, offset: uint32)` pairs | ~65 KB |

Font glyphs remain in Flash at runtime and are read on demand.

## 4. API Reference

### 4.1 `mie::TrieSearcher` Рђћ `include/mie/trie_searcher.h`

```cpp
/// Single search result.
struct Candidate {
    char     word[32];  // UTF-8, null-terminated; up to 10 CJK chars
    uint16_t freq;      // frequency weight (higher = more common)
};

class TrieSearcher {
public:
    /// Load from disk files (PC / host mode).
    /// Returns false if a file cannot be opened or the header is malformed.
    bool load_from_file(const char* dat_path, const char* val_path);

    /// Load from in-memory buffers (embedded / PSRAM mode).
    /// The buffers must remain valid for the lifetime of this object.
    bool load_from_memory(const uint8_t* dat_buf, size_t dat_size,
                          const uint8_t* val_buf, size_t val_size);

    /// Look up a Bopomofo phoneme sequence.
    /// Writes up to max_results candidates into out[], sorted by frequency.
    /// Returns the number of candidates written (0 if not found or not loaded).
    int search(const char* key_utf8, Candidate* out, int max_results) const;

    bool     is_loaded()  const;
    uint32_t key_count()  const;
};
```

**Thread safety:** all methods are read-only after `load_from_*()`; concurrent
reads from multiple threads are safe without external locking.

**Phase 2 note:** the `load_from_memory()` path maps directly onto PSRAM
pointers.  The binary format and API are frozen at this revision.

### 4.2 `mie::ImeLogic` Рђћ `include/mie/ime_logic.h`

```cpp
enum class InputMode : uint8_t {
    Bopomofo     = 0,   // consonant-first Bopomofo prediction (Phase 1 active)
    English      = 1,   // English word prediction             (Phase 3)
    Alphanumeric = 2,   // traditional multi-tap               (Phase 3)
    Calculator   = 3,   // calculator UI                       (Phase 3)
};

class ImeLogic {
public:
    explicit ImeLogic(TrieSearcher& searcher);

    /// Feed one key event from the HAL.
    /// Returns true if the display should be refreshed.
    bool process_key(const KeyEvent& ev);

    InputMode        mode()            const;  // current input mode
    const char*      input_str()       const;  // accumulated phoneme sequence (UTF-8)
    int              input_bytes()     const;
    int              candidate_count() const;
    const Candidate& candidate(int i)  const;  // 0-based, unchecked

    void clear_input();
};
```

**Key event routing (Bopomofo mode):**

| Physical key | Row, Col | Action |
|---|---|---|
| Phoneme keys (rows 0РђЊ3, cols 0РђЊ4) | see ┬Д5 | Append primary phoneme; run search |
| `BACK` | 2, 5 | Remove last UTF-8 code point; run search |
| `DEL` | 3, 5 | Same as BACK |
| `MODE` | 4, 0 | Cycle input mode; clear input |
| `SPACE` | 4, 2 | Commit first candidate; clear input |
| `OK` | 5, 4 | Commit first candidate; clear input |
| All other keys | Рђћ | Ignored in Phase 1 |

### 4.3 `mie::IHalPort` Рђћ `include/mie/hal_port.h`

```cpp
/// A single key event delivered from the platform HAL to IME-Logic.
struct KeyEvent {
    uint8_t row;      // matrix row   (0РђЊ5)
    uint8_t col;      // matrix column (0РђЊ5)
    bool    pressed;  // true = key down, false = key up
};

/// Abstract HAL interface.  Every platform must provide a concrete subclass.
class IHalPort {
public:
    virtual ~IHalPort() = default;

    /// Non-blocking.  Returns true and fills out if a key event is pending.
    virtual bool poll(KeyEvent& out) = 0;
};
```

### Generating dictionary data (requires MoE CSV)

```bat

    --moe-csv moe_dict.csv ^
    --output-dir firmware\mie\data
```

Expected CSV columns (UTF-8 with BOM):

| Column | Content |
|--------|---------|
| `Т│ежЪ│` | Bopomofo reading (may span multiple syllables) |
| `УЕъУфъ` | Word in Traditional Chinese |
| `жа╗ујЄ` | Integer frequency weight (defaults to 1 if absent) |

Output: `dict_dat.bin`, `dict_values.bin`, `dict_meta.json`.

### Generating font data (requires GNU Unifont)

```bat
python firmware\mie\tools\gen_font.py ^
    --unifont unifont-15.1.04.hex ^
    --charlist charlist_8104.txt
```

Download GNU Unifont from <https://unifoundry.com/unifont/>.
`charlist_8104.txt` must contain one Unicode code point (hex) per line for
each of the 8,104 MoE standard characters.

### Running the REPL

```bat
:: Without dictionary (keys shown by name only)
build\mie-host\mie_repl.exe

:: With dictionary (shows live candidates)
build\mie-host\mie_repl.exe ^
    --dat firmware\mie\data\dict_dat.bin ^
    --val firmware\mie\data\dict_values.bin
```

**REPL controls:**

| Key | Action |
|-----|--------|
| Phoneme keys (`1 3 5 7 9 q e t u o a d g j l z c b m \`) | Append Bopomofo phoneme |
| `Backspace` | Remove last phoneme |
| `` ` `` (MODE) | Cycle input mode; clear input |
| `Space` / `Enter` | Commit first candidate; clear input |
| `Ctrl-C` / `ESC` | Quit |

**Example session** Рђћ typing `e` `j` `u` produces the phoneme sequence
сёЇ сёе сёД and should yield candidates such as УдЈ / ТГИ / УЕГ when the
dictionary is loaded.
